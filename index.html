ML Methodology

How the Model Was Built

A fully transparent ML pipeline — from raw data collection to a trained scikit-learn Random Forest, served by Flask.

Training Pipeline (run train.py to reproduce)

1

Data Collection

1,613 labeled domains across 4 classes. Legitimate domains from Nepal's .np zone + Tranco top sites. Phishing domains generated to mirror PhishTank statistical properties (brand names + phishing keywords + high-abuse TLDs). Malware/DGA domains simulate real DGA algorithms (high entropy, low vowel ratio, digit-heavy). Spam domains mirror SURBL patterns.

2

Feature Extraction (features.py)

For each domain, extract_features() computes 22 numerical values — Shannon entropy, Levenshtein distance, TLD abuse score from Spamhaus data, phishing keyword score from PhishTank vocabulary, n-gram matching, vowel deviation, and more. Every domain becomes a vector of 22 numbers.

3

Train / Test Split

80% of data (1,290 domains) used for training. 20% (323 domains) held out for evaluation. Stratified split ensures all 4 classes are proportionally represented in both sets.

4

StandardScaler

Features are normalized so large-magnitude features (like domain_length=25) don't dominate small ones (like digit_ratio=0.1). The scaler is fit only on training data, then applied to test data — preventing data leakage.

5

Random Forest (n_estimators=200)

200 decision trees are trained, each on a random subset of data and features. Each tree votes on the class. The majority vote wins. This ensemble approach is more robust than a single decision tree and gives feature importances automatically — showing exactly which features matter most.

6

Evaluation

Accuracy: 98.5% on held-out test set. 5-fold cross-validation: 98.8% ± 0.5%. The model is then saved with pickle to models/rf_model.pkl and loaded by app.py on startup.

📊

Shannon Entropy

Measures randomness. DGA botnet domains are algorithmically generated — high entropy, unpronounceable. Natural human words have low entropy.

H = -Σ p(x)·log₂p(x)
📐

Levenshtein Distance

Edit-distance to known brand names. If a domain is 1–2 edits away from "esewa" or "nabilbank", it's almost certainly typosquatting.

dist("g00gle","google")=2
🌐

TLD Abuse Score

Each TLD is scored by historical abuse rate from Spamhaus data. .tk = 97%, .ml = 95%, .gov.np = 0%. Single strongest predictor in the model.

score(".tk") = 0.97
🔤

Vowel Deviation

Natural English words are ~38% vowels. DGA domains deviate significantly. Feature = |vowels/letters − 0.38|. Key DGA detection signal.

|vowels/letters − 0.38|
🎣

Phishing Keywords

Domain scored against a weighted vocabulary of 27 phishing terms from PhishTank analysis. "login+secure+verify" in one domain = very high score.

"login" → weight 0.25
🔣

N-gram Matching

Extracts all 4-character substrings and checks against known phishing n-grams. "logi", "secu", "veri", "bank" are common in phishing domain names.

"nabillogin" → "logi" hit
